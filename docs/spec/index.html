仕様と特徴

デザインポリシー

glas は、モダンなCLIツールである eza にインスパイアされつつ、Gitとの緊密な統合を目指して作られています。

  • 状態とロジックの分離: ファイルのメタデータ取得、Gitステータスの取得、および端末への描画ロジックが独立して設計されています。
  • パフォーマンス最適化: パス走査時のアロケーションを抑え、数万規模のファイルを含むリポジトリでも軽快に動作します。
  • マルチプラットフォーム互換性: Windows, macOS, Linux すべての環境で動作し、各OS固有のファイル属性（NTFSの隠しファイル属性など）に対応しています。

挙動の優先順位と特別仕様

  • インデックスセレクタの自動変換: --cached または --stage オプションを指定した場合、Gitインデックス（ファイル単位の情報）から読み出すため、自動的に -f (--only-files) が有効になり、空のディレクトリ等は表示対象から除外されます。
  • ファイルの個別指定時のカラーリング: glas Cargo.toml のようにファイルを直接指定して呼び出した場合でも、そのファイルのGit上のステータス（変更中＝黄色など）が適用されて出力されます。
  • README / 特殊ファイルの強調表示: Cargo.toml や README.md, justfile などのプロジェクトにおける重要な特殊ファイルは、Gitステータスに関わらず常に太字で表示され、一目で認識できるようになっています。
